# I18n (国際化) ガイドライン

Rails の I18n 管理に関するルールとパターン。

---

## 基本方針

- ロケールファイル: `config/locales/ja.yml`
- デフォルト言語: 日本語 (ja)
- すべてのモデル属性は ActiveRecord の i18n に定義する

---

## モデル・カラム追加時のチェックリスト

新しいモデルやカラムを追加した際は、**必ず** `config/locales/ja.yml` を更新する。

### 新規モデル追加時

```yaml
ja:
  activerecord:
    attributes:
      model_name:        # スネークケースで追加
        attribute1: "属性1の日本語名"
        attribute2: "属性2の日本語名"
```

### 既存モデルにカラム追加時

該当モデルのセクションに属性を追記する。

---

## 命名パターン

| 属性タイプ | 日本語表記例 |
|-----------|-------------|
| email | メールアドレス |
| name | 名前 |
| password | パスワード |
| created_at | 作成日時 |
| updated_at | 更新日時 |

---

## 現在の構造例

```yaml
ja:
  activerecord:
    attributes:
      admin:
        email: "メールアドレス"
        name: "名前"
      employee:
        email: "メールアドレス"
        name: "名前"

  custom_errors:
    controllers:
      # エラーメッセージ定義
```

---

## 注意事項

- マイグレーション作成後、ja.yml の更新を忘れない
- バリデーションエラーメッセージが正しく日本語表示されるか確認する
- `rails console` で `ModelName.human_attribute_name(:attribute)` を使って確認可能
